🤖 Agent Profiles — Functions, Tools & Triggers
🧠

OrchestratorAgent

Master Router & Admin Q&A
AGENT 1

First point of contact for all admin interactions. Routes incoming tasks to the correct specialist agent, synthesizes multi-agent responses, answers admin questions using RAG over live church data, and delivers a morning daily briefing of everything that needs attention.

  • handoff_to_agent(agent_name, task)Delegates task to specialist agent
  • query_knowledge_base(question)RAG over church data for Q&A
  • get_system_status()Overall snapshot across all agents
  • list_pending_tasks()What needs attention right now
  • generate_daily_briefing(church_id)Morning summary report for admin
  • search_members(query)Quick member lookup by name/ID
  • get_church_overview(church_id)Full church stats summary
  • SCHEDULEDEvery morning at 7:00 AM — daily briefing delivered to platform admin
  • ON-DEMANDAdmin types any question in the chat interface
  • ESCALATIONAny agent escalates an urgent issue to Orchestrator
  • WEBHOOKPaystack payment event received — route to SubscriptionWatchdog
  • Q&A"How many members joined last month?"
  • Q&A"Which churches are about to expire?"
  • Q&A"What is the total income for March?"
  • Q&A"Is there anything urgent I need to handle today?"
  • Q&A"Show me all churches on the TRIAL plan"
  • Q&A"Send birthday greetings to all members today"
🔔

SubscriptionWatchdogAgent

Account Expiry & Payment Guardian
AGENT 2

Monitors all church subscription statuses across every tenant. Detects trial expirations, payment failures, and lapsed subscriptions. Sends automated tiered reminder emails and SMS. Disables church access post-expiry. Generates daily subscription health reports for the platform admin.

  • get_expiring_subscriptions(days_ahead)Query churches by expiry date
  • get_failed_payments()Pull Paystack failed transactions
  • send_renewal_reminder_email(church_id, days_left)Templated expiry warning
  • send_sms_alert(church_id, message)Twilio/mNotify SMS to church admin
  • disable_church_access(church_id)Auto-lock expired church
  • reinstate_church_access(church_id)Re-enable after payment confirmed
  • generate_subscription_report()Full status across all tenants
  • notify_support_team(alert)Escalate critical issues to support
  • get_payment_history(church_id)Full Paystack payment log
  • SCHEDULEDEvery morning at 7:00 AM — full expiry scan
  • WEBHOOKPaystack fires on payment failure → immediate alert
  • WEBHOOKPaystack payment success → auto-reinstate access
  • ON-DEMAND"Show me all churches expiring this week"
  • EVENTTrial plan created → start 7d countdown
  • D-7"Your trial expires in 7 days — upgrade now"
  • D-3"Action needed: your plan expires in 3 days"
  • D-1"Last chance — access expires tomorrow"
  • D-0"Your account has been suspended"
  • FAIL"Payment failed — please update your billing info"
  • OK"Payment confirmed — your subscription is active"
💰

TreasuryHealthAgent

Financial System Monitor & Analyst
AGENT 3

Monitors treasury activity across all church tenants. Detects anomalies like unusually large transactions and budget overruns. Alerts treasurers on stalled expense approvals. Generates financial summaries and answers finance questions from the admin on demand.

  • get_income_summary(church_id, date_range)Income totals by category
  • get_expense_summary(church_id, date_range)Expense breakdown per department
  • get_stalled_expense_requests(days_stalled)Overdue approval requests
  • detect_anomalies(church_id)Flag unusual/large transactions
  • get_budget_vs_actual(church_id, dept_id)Budget utilization per department
  • send_treasurer_alert(church_id, message)Email/SMS to church treasurer
  • generate_financial_report(church_id, format)PDF/XLSX export
  • get_asset_inventory(church_id)Full church asset list
  • get_treasury_statistics(church_id)KPI dashboard data
  • SCHEDULEDEvery Monday at 8:00 AM — weekly financial health check
  • EVENTExpense request sits in approval stage > 48 hours
  • EVENTTransaction recorded above anomaly threshold
  • SCHEDULEDMonthly — budget vs. actual report for all departments
  • ON-DEMAND"What is the financial health of Church X?"
  • STALLExpense request pending > 48h → nudge to approver
  • BUDGETDepartment at 80% budget utilization → warn dept head
  • ANOMALYLarge transaction above threshold → alert treasurer + platform admin
  • INACTIVENo income recorded in 2 weeks → flag church as inactive
  • ASSETAsset depreciation review due → reminder to treasury

MemberCareAgent

Member Engagement & Lifecycle Manager
AGENT 4

Manages the entire member lifecycle — from visitor follow-up through active engagement. Sends personalized birthday greetings, welcome emails for new members, flags inactive members, and follows up with unconverted visitors. Monitors membership status changes and suggests next steps.

  • get_members_with_birthdays(date)Birthday lookup for today/this week
  • get_unconverted_visitors(days_since_visit)Visitor follow-up list
  • get_inactive_members(days_inactive)Lapsed/disengaged members
  • get_new_members(date_range)Recently joined members
  • send_birthday_email(member_id)Personalized birthday greeting
  • send_visitor_followup(visitor_id)Follow-up outreach template
  • send_welcome_email(member_id)New member onboarding email
  • convert_visitor_to_member(visitor_id, data)API call to convert record
  • get_member_profile(member_id)Full member details
  • query_member_stats(church_id)Aggregate membership statistics
  • SCHEDULEDDaily at 8:00 AM — birthday and visitor follow-up scan
  • SCHEDULEDEvery Sunday evening — inactive member scan (> 30 days)
  • EVENTNew member created → send welcome email immediately
  • EVENTMember status changes to TRANSFER → notify agent
  • ON-DEMAND"Send a message to all inactive members"
  • BDAY"Happy Birthday [Name]! We celebrate you today"
  • D+3Visitor: "Great to meet you — here's what we offer"
  • D+7Visitor: "We'd love to have you join our family"
  • NEW"Welcome to [Church Name]! Here's how to get started"
  • D+30Inactive: "We miss you! Come back and connect with us"

DepartmentProgramAgent

Department Operations & Workflow Monitor
AGENT 5

Monitors program approval workflows for stalled steps and reminds department heads of pending actions. Tracks department activity schedules and sends reminders. Monitors budget allocation vs. spending per department. Answers questions about department structure and program status.

  • get_stalled_programs(days_stalled)Programs stuck in an approval step
  • get_upcoming_activities(church_id, days_ahead)Upcoming activity reminders
  • get_department_budget_status(church_id)Budget utilization per dept
  • notify_department_head(dept_id, message)Email/SMS to department head
  • get_pending_program_approvals(church_id)All programs awaiting action
  • get_department_members(dept_id)Member list for a department
  • send_activity_reminder(activity_id)Pre-event reminder to organizer
  • get_program_details(program_id)Full program info including budget items
  • get_program_approval_history(program_id)Approval trail with timestamps
  • SCHEDULEDDaily scan for programs stalled > 72 hours
  • SCHEDULED24 hours before any department activity → send reminder
  • EVENTProgram step unsigned > 72h → nudge next approver
  • SCHEDULEDMonthly — department activity report
  • ON-DEMAND"Which programs are waiting for approval?"
  • STALLProgram approval stalled > 72h → nudge to next approver
  • REMINDActivity tomorrow → reminder to organizer + participants
  • BUDGETOverspend risk detected → warn department head
  • IDLENo department activities in 30 days → flag to admin

AnnouncementAgent

Communications & Publishing Coordinator
AGENT 6

Monitors announcements stuck in PENDING_REVIEW. Distributes published announcements to relevant members and departments via email or SMS. Reminds approvers of pending items. Generates weekly announcement digest summaries and provides statistics on reach and engagement.

  • get_pending_announcements(church_id)Announcements awaiting approval
  • get_published_announcements(church_id, range)Published announcements list
  • approve_and_publish(announcement_id)Approve and publish (if authorized)
  • send_announcement_to_members(ann_id, group)Blast email/SMS to target group
  • notify_approver(announcement_id)Nudge reviewer with reminder
  • generate_weekly_digest(church_id)Weekly summary email
  • get_announcement_stats(church_id)Reach and engagement analytics
  • get_announcement_categories(church_id)List available categories
  • SCHEDULEDEvery 6 hours — check for pending announcements
  • SCHEDULEDEvery Sunday morning — generate weekly digest
  • EVENTAnnouncement published → auto-distribute to members
  • EVENTAnnouncement in PENDING_REVIEW > 24h → nudge approver
  • ON-DEMAND"Send this week's announcements to all members"
  • DIGESTWeekly announcement digest to all subscribed members
  • PUBLISHImmediate distribution on announcement publish
  • REMINDPending review > 24h → nudge to approver
  • URGENTURGENT priority announcement → immediate blast

AuditSecurityAgent

Compliance, Audit & Security Monitor
AGENT 7

Scans audit logs for suspicious activity including failed logins, bulk deletes, and unauthorized permission changes. Monitors for unusual access patterns. Flags account lockouts. Generates compliance reports. Cross-references with TreasuryHealthAgent to flag suspicious financial activity.

  • get_audit_logs(church_id, action_type, range)Full audit history
  • get_failed_login_attempts(threshold)Brute force detection
  • get_permission_changes(church_id, range)RBAC changes audit
  • get_locked_accounts(church_id)Users locked after 5 failed logins
  • flag_suspicious_activity(church_id, details)Alert support team
  • generate_audit_report(church_id, range)Compliance PDF report
  • send_security_alert(admin_email, message)Urgent email to platform admin
  • detect_bulk_actions(church_id, threshold)Flag mass deletes/edits
  • REAL-TIMEAccount locked after 5 failed logins → immediate alert
  • SCHEDULEDNightly at 1:00 AM — full audit log scan
  • EVENT10+ records deleted in 5 minutes → flag bulk action
  • EVENTUnusual access at odd hours → flag suspicious login
  • ON-DEMAND"Show all security events for Church X this week"
  • LOCKOUTAccount locked after 5 failed attempts
  • BULKBulk delete / mass action flagged
  • PERMUnauthorized permission change detected
  • LOGINLogin from unusual IP / odd hours
  • REPORTMonthly compliance report to platform admin

SecretariatAgent

Official Records, Documents & Correspondence
AGENT 8

Manages official church records and formal correspondence. Generates transfer letters, membership certificates, and recommendation letters. Records meeting minutes, baptism records, and marriage records. Reminds secretary of pending document requests. Answers admin questions about official church records.

Note: The secretariat Django app is currently an empty scaffold. Models (MeetingMinutes, DocumentRequest, BaptismRecord, TransferLetter, CorrespondenceLog) must be built before this agent can be fully operational.

  • log_baptism_record(member_id, date, officiant)Sacrament record

  • get_pending_document_requests()Outstanding document requests
  • send_document_by_email(member_id, doc_type)Deliver document via email
  • notify_secretary(message)Escalate to human secretary
  • ON-DEMAND"Generate a transfer letter for [Member Name]"

  • SCHEDULEDWeekly reminder of pending document requests

⚙️ Implementation Stack

Agent Framework

OpenAI Agents SDK
Lightweight tools + handoffs between agents. Clean, minimal code.

Stateful Workflows

LangGraph
Multi-step approval monitoring and stateful agent workflows with persistence.

Multi-Agent Crews

CrewAI
Report generation tasks and coordinated multi-agent output.

Tool Exposure Layer

MCP Server
Exposes all 50+ Django REST API endpoints as agent tools cleanly.

Task Scheduling

Celery Beat
Already in your Django system. Triggers agents on defined schedules.

Agent Memory

SQLite + Redis
SQLite for conversation memory. Redis for ephemeral session state.

Admin Q&A (RAG)

ChromaDB / Pinecone
Vector database for semantic search over church data and documents.

LLM Provider

OpenAI GPT-4.1
Primary model for all agents. Can swap per agent based on task complexity.
🔌 How Agents Connect to Your Django System

An MCP Server wraps your existing Django REST API, exposing every endpoint as a callable tool. Agents never touch the database directly — they call tools, which call your API.
