OpenAI

LLM brain for all 8 agents
MUST CREATE

The core intelligence behind every agent. All reasoning, Q&A, email drafting, document generation and decision-making runs through GPT-4.1.

  • 1Go to platform.openai.com → Sign up / Log in
  • 2Go to API Keys → Create new secret key
  • 3Add billing — set a monthly spend limit (e.g. $20)
  • 4Save key securely — it is shown only once
  • OPENAI_API_KEYsk-proj-... (primary agent key)
  • OPENAI_MODELgpt-4.1 (or gpt-4.1-mini to save cost)
  • OPENAI_ORG_IDOptional — if using an org account
🔭

LangSmith

Agent tracing, debugging & monitoring
RECOMMENDED

Traces every step each agent takes — what it reasoned, which tool it called, what it returned. Invaluable for debugging when agents do unexpected things.

  • 1Go to smith.langchain.com → Sign up (free tier available)
  • 2Create a new Project called church-agents
  • 3Settings → API Keys → Create key
  • LANGCHAIN_TRACING_V2true
  • LANGCHAIN_API_KEYls__... your key
  • LANGCHAIN_PROJECTchurch-agents
🧩

Pinecone or ChromaDB

Vector DB for admin Q&A (RAG)
MUST CREATE

Powers the OrchestratorAgent's ability to answer admin questions about church data. Stores embedded chunks of member records, treasury data, and announcements.

Recommended: Start with ChromaDB locally (free, no account needed), then upgrade to Pinecone for production.

  • 1Local (dev): pip install chromadb — no account needed
  • 2Production: Go to pinecone.io → Sign up (free starter)
  • 3Create an index called church-knowledge, dimension 1536
  • 4API Keys → copy your key
  • PINECONE_API_KEYProduction vector store key
  • PINECONE_INDEX_NAMEchurch-knowledge
  • VECTOR_STORE_TYPEchroma (dev) or pinecone (prod)

Paystack

Subscription billing & webhooks
VERIFY WEBHOOK

Already configured for payments. However, the SubscriptionWatchdogAgent needs the Paystack webhook properly registered to receive real-time payment failure/success events.

  • 1Log in to dashboard.paystack.com
  • 2Settings → API Keys & Webhooks
  • 3Set webhook URL to: https://yourdomain.com/api/webhooks/paystack/
  • 4Enable events: charge.success, charge.failed, subscription.disable
  • PAYSTACK_SECRET_KEY✓ Already in settings.py
  • PAYSTACK_PUBLIC_KEY✓ Already in settings.py

6️⃣ Create a Dedicated Agent Bot User in Your Django System

Your MCP server needs a JWT token to call the Django API. Create one dedicated platform-admin user specifically for the agents — never use a real person's credentials.

# Run this in your Django backend to create the agent bot user # Option 1 — Django shell python manage.py shell >>> from accounts.models import User, Church >>> user = User.objects.create_superuser( ... email="agent-bot@yourdomain.com", ... password="use-a-very-strong-password-here", ... first_name="Church", ... last_name="Agent Bot", ... is_platform_admin=True, # needs access to all churches ... is_active=True ... ) # Option 2 — Management command (create this once) # management/commands/create_agent_user.py # python manage.py create_agent_user
